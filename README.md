## ✨ 核心特性

- **备份导入导出** - 支持备份的导入和导出功能

## 📖 使用指南

### 1. 添加 API 供应商

1. 点击 **"添加 Provider"** 按钮
2. 选择分类：**国内 API** / **国际 API** / **本地/局域网**
3. 选择供应商（如 Moonshot、OpenAI 等）
4. 填写 **API 地址** 和 **API Key**
5. 选择要使用的**模型**
6. 点击 **"保存配置"**

### 2. 切换模型

1. 在 **Providers** 标签页查看已配置的供应商
2. 点击想要使用的 **Provider 卡片**
3. 在弹出的模型列表中选择模型
4. 点击 **"应用配置"**
5. 配置自动同步到 OpenClaw

### 3. 诊断工具

1. 切换到 **诊断** 标签页
2. 点击 **"运行系统检查"** 进行全面诊断
3. 查看检查结果，根据提示修复问题
4. 使用 **"一键修复 Gateway"** 自动修复服务问题

### 4. 备份管理

1. 在 **Providers** 标签页点击 **"管理备份"**
2. 查看所有备份列表
3. 点击 **"恢复"** 还原到之前的配置
4. 点击 **"导出"** 将备份保存到本地

## 🛠️ 故障排除

### 常见问题

#### 1. 启动时提示缺少依赖
**解决方案**: 程序会自动安装依赖，请等待安装完成。如果失败，手动运行：
```bash
npm install
```

#### 2. Gateway 服务无法启动
**解决方案**: 
1. 点击顶部 **"修复"** 按钮
2. 程序会自动重新安装并启动 Gateway
3. 等待 1-2 分钟完成修复

#### 3. API 连接测试失败
**解决方案**:
1. 检查 API Key 是否正确
2. 确认 Base URL 是否可访问
3. 查看 **诊断** 标签页的详细错误信息

#### 4. 配置未同步到 OpenClaw
**解决方案**:
1. 确认 OpenClaw 已正确安装
2. 检查 OpenClaw 配置文件权限
3. 使用 **诊断** 工具检查配置完整性
